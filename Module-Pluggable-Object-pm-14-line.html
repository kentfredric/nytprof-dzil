254

354Module::Pluggable::Object - automatically give your module the ability to have plugins
355
356=head1 SYNOPSIS
357
358
359Simple use Module::Pluggable -
360
361 package MyClass;
362 use Module::Pluggable::Object;
363
364 my $finder = Module::Pluggable::Object->new(%opts);
365 print "My plugins are: ".join(", ", $finder->plugins)."\n";
366
367=head1 DESCRIPTION
368
369Provides a simple but, hopefully, extensible way of having 'plugins' for
370your module. Obviously this isn't going to be the be all and end all of
371solutions but it works for me.
372
373Essentially all it does is export a method into your namespace that
374looks through a search path for .pm files and turn those into class names.
375
376Optionally it instantiates those classes for you.
377
378This object is wrapped by C<Module::Pluggable>. If you want to do something
379odd or add non-general special features you're probably best to wrap this
380and produce your own subclass.
381
